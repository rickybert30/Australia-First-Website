Explore the data with SQL

Run your own SQL queries over the whole dataset — candidates, positions, donors and donations — right in your browser. Nothing is sent to a server; the database is downloaded once and queried locally. ← Back to the database

This is the same data behind the site, as a relational SQLite database. Tables: candidate, position, donor, donor_info, party_donation, member_donation, and a deduplicated source. The donor.canonical_id column links spelling variants of the same donor to one entity — use COALESCE(canonical_id, name) to roll them up. Queries are read-only.
